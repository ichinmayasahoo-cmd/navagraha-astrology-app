# Navagraha — Vedic Astrology & Numerology Web App

A full-stack app that takes a person's **name, date of birth, time of birth, and place of birth**, and returns:

- **Mulank** (root number) and **Bhagyank** (destiny number) — pure numerology, calculated with plain math.
- A **real astronomical natal chart**: ascendant (lagna), moon sign (rashi), and every major planet's sign, house, degree, and retrograde status — calculated with an actual ephemeris library (sidereal zodiac, whole-sign houses, the standard Vedic setup).
- An AI-written **reading** (personality, love & relationships, career & success, strengths/challenges, practical guidance) generated by the Claude API, but only after being given the real computed chart — Claude is instructed to interpret the data, never invent it.

## Why this architecture

Planetary positions can't be reliably "guessed" by a language model — they require real astronomical calculations (ephemeris data, timezone handling, sidereal offsets, house systems). This app:

1. Geocodes the birth place → latitude/longitude (OpenStreetMap Nominatim, free, no key).
2. Computes the actual chart from lat/long + date/time using an ephemeris-based library (`circular-natal-horoscope-js`), which also auto-resolves the correct historical timezone/DST for that place and moment.
3. Computes numerology with simple digit-sum math.
4. Sends **only the computed chart + numbers** (not the raw birth data) to Claude, asking it to write the human-readable interpretation grounded in that data.

## Setup

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Add your Anthropic API key**
   ```bash
   cp .env.example .env
   # then edit .env and paste your key
   ```
   Get a key at https://console.anthropic.com (Claude API / Claude Platform).

3. **Run it**
   ```bash
   npm start
   ```
   Then open http://localhost:3000

## Notes & things to customize before going to production

- **Geocoding rate limits**: Nominatim's free tier asks for ≤1 request/second and a descriptive User-Agent (already set in `src/geocode.js`). For real traffic, swap in Google Geocoding, Mapbox, or LocationIQ.
- **Model**: `src/claudeClient.js` uses `claude-sonnet-5`. Check https://docs.claude.com for the current recommended model ID if you're reading this much later.
- **House system / zodiac**: currently `sidereal` zodiac + `whole-sign` houses (classic Vedic/Jyotish defaults). If you want Western tropical charts instead, change `zodiac: 'tropical'` and pick a house system like `placidus` in `src/astrology.js`.
- **Disclaimers**: the reading always ends with a short "for reflection/entertainment" note — keep this, astrology/numerology readings should not be presented as guaranteed fact.
- **Numerology system**: the Mulank/Bhagyank formulas here reduce every number fully to 1–9 (no master numbers like 11/22 kept). Some traditions differ slightly — adjust `src/numerology.js` if you follow a specific school.
- **CORS**: currently wide open (`cors()`); restrict `origin` before deploying publicly.
